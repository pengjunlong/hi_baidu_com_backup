OpenFlow-1.0-installation.

Dependencies

OpenFlow requires some dependencies. To install them:

sudo apt-get install git-core automake m4 pkg-config libtool

sudo apt-get install gcc make libc6-dev autoconf autotools-dev

 

Downloads

Download Openflow 1.0 at our download page and unzip it or download it via VLI:

wget http://www.gta.ufrj.br/omni/phocadownload/openflow-1-0.tar.gz
tar -xzvf openflow-1-0.tar.gz -C /root

 

To install OpenFlow from the Stanford Git repository:

git clone git://openflow.org/openflow.git

 

Installation

After OpenFlow has been downloaded:

cd openflow
./boot.sh

 

To build OpenFlow, inside the OpenFlow directory:

./configure
make
sudo make install

 

Initialization

In order to start an OpenFlow switch in your computer, run the following commands (interfaces, in the list of interfaces, must be separated by commas, and name is the name of a file used to control the OpenFlow switch, the & is to keep this command running in background):

ofdatapath -i <list of interfaces> punix:<name> &

 

After that, connects the switch in a controller by the command:

ofprotocol unix:<name> tcp:<controller address>:<controller port>

For more information about OpenFlow, please visit openflow.org.
