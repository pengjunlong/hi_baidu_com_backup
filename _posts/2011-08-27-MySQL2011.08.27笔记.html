视图里面的数据什么时候准备好.

用到视图的时候，MYSQL视图不包含任何数据，只不过是个SQL语句。

 

 

MYSQL的存储程序都有这个问题，就是在创建的时候，并不会去检查逻辑结构。只会检查语法，语法通过就能够创建成功，实际执行的时候才会检查逻辑，比如某个字段不存在啥的，在执行的时候才会给你报错。

 

 

如何把更符合条件的排在前面呢？例如符合ABC的排在最前，符合AB或BC、AC的排在随后呢？

select * from table where cn like'%A%' or cn like'%B%' or cn like'%C%'  
order by (cn like'%A%') + (cn like'%B%') + (cn like'%C%' ) desc，
(cn like'%A%') + (cn like'%B%') desc,
(cn like'%B%') + (cn like'%C%' ) desc,
(cn like'%A%') + (cn like'%C%' ) desc,
(cn like'%A%') desc,
(cn like'%B%') desc,

(cn like'%C%' ) desc

可以借助 excel 等来生成 1000 个 ID 嘛
要是 dos 下，可如下生成：
SQL code
@echo offsetlocal enabledelayedexpansionfor /L %%i in (1,1,1000) do ( set num=0000%%i set num=!num:~-4! echo 10000000000!num!)

SQL code
seq 1 1000
